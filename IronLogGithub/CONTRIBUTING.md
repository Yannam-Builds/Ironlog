# Contributing to IronLog

Thanks for contributing.

## Quick Rules
- Keep changes focused and minimal.
- Avoid unrelated refactors in the same PR.
- Add/update tests when logic changes.
- Keep UX consistent with existing style.

## Workflow
1. Fork and create a feature branch.
2. Make changes with clear commit messages.
3. Run local checks/build before opening PR.
4. Open PR with:
   - summary
   - screenshots (if UI)
   - testing notes

## PR Checklist
- [ ] Builds locally
- [ ] No obvious regressions
- [ ] Updated docs/changelog if needed
